## Task

Redesign dashboard agent summary tables to replace numeric count columns with an activity timeline visualization.

## Summary

Replaced low-signal Today/7d/30d alert count columns with a single "Recent" column showing the last 3 activity results as colored badge mini-timelines (oldest→newest). Removed top summary cards for alert counts while retaining Symbols Watched and Open Positions metrics.

## Design Decisions

### Decision 1: Replace counts with activity timeline badges
**Rationale:** A count of "3 alerts" doesn't indicate severity or actionability. Seeing `[WAIT] › [WAIT] › [SELL]` immediately shows user intent and pattern. Badges reuse existing styling; no new CSS classes needed.

**Alternative considered:** Keep counts but add inline badge preview. **Rejected:** Takes same space, less usable.

### Decision 2: Limit to 3 recent activities
**Rationale:** Balances pattern visibility (3 results show trend) with table density. Users can click badge to see full activity feed.

### Decision 3: Keep `grand_totals` in backend but don't render
**Rationale:** Future features may use these counts (e.g., historical comparison charts). Removes UI clutter while preserving data pathway.
